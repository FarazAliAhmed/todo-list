# Contributing to Evolution of Todo

Thank you for your interest in contributing to this project!

## Development Setup

1. Clone the repository
2. Install dependencies: `npm run install:all`
3. Set up environment variables (see README.md)
4. Run development servers: `npm run dev`

## Project Structure

- `frontend/` - Next.js web application
- `backend/` - FastAPI server
- `src/` - Phase I console application (legacy)
- `.kiro/specs/` - Specification documents

## Development Workflow

This project follows spec-driven development:

1. **Requirements** - Define what needs to be built
2. **Design** - Plan the technical implementation
3. **Tasks** - Break down into actionable steps
4. **Implementation** - Build according to specs

See `.kiro/specs/phase2-fullstack-web/` for current phase specifications.

## Code Style

- **Frontend**: Follow TypeScript and React best practices
- **Backend**: Follow PEP 8 Python style guide
- **Commits**: Use clear, descriptive commit messages

## Testing

- Write tests for new features
- Ensure all tests pass before submitting
- Frontend: Component and integration tests
- Backend: Unit and API tests

## Pull Request Process

1. Create a feature branch
2. Make your changes
3. Write/update tests
4. Update documentation
5. Submit pull request with clear description

## Questions?

Open an issue for discussion or clarification.
